# c-programming-modern-approach

## Chapter 1: History of C

- C is the product of the UNIX operating system
- UNIX: Developer at Bell Labs in assembly language - 1970s
- Multiuser, multi-tasking system.
  - Programs can be combined using **pipes** and **shell** scripts to perform complex tasks.
  - Written in **C** makes it portable to different machines.
  - All devices and files are represented in a single tree-like directory structure.
  - Access to files and processes is controlled through user and group permissions.
  - Grandfather for modern OS system design.
- **C** Standardization Timeline:
  - American National Standards Institute (ANSI): ANSI C
    - Standardized **C** in 1989-90 -> C89
  - ISO approved in 1990 -> C90
    - Added new types -> C99
- Underlying Philosophy
  - C is low-level language
  - C is small language
  - C is flexible
  - C is portable
  - C has large standard library
  - Integration with UNIX
  - Relatively difficult to understand and modify.
  - C programs are error prone
- **LINT**
  - Error checker tool other than the C compiler
  - Traditionally provided in UNIX
  - Performs more extensive analysis than most C compilers
- **SPLINT**
  - Enhanced version of LINT (Secure Programming Lint)
  - Included in most LINUX operating systems.
